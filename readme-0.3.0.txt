部署说明
=======

服务启动
-------
在服务器上请使用tmux等终端复用器
用最新版本的jar包替换原来的jar包，把原来的sql文件夹删除（不要留有旧的sql文件），放上新的sql文件夹
进入jar包所在目录，不要更改目录结构，执行：
java -jar streamserver-0.3.0.jar

日志流和查询启动
---------------
目前启动后并没有自动创建日志流，需要点击首页左上角的[样例演示]按钮才能创建要演示的日志流和查询，点击后请等待两秒然后刷新，便可以看到日志流列表。
点击日志流列表可以进入到查询管理界面，点击上方[折叠全部]按钮可以折叠所有查询，然后可以点击每个查询条目打开查询

查询可视化
---------
自动判断是使用图还是表进行可视化，如果默认的可视化方法不好，可以将鼠标移动到横栏上选择[新增图表]，然后选择是图还是表。比如频繁序列默认使用表进行可视化，但是如果改成图这些频繁序列的频次对比会更加清楚。

序列预测功能
------------
选择[新增,系统管理员]可以看到下一步是[删除,系统管理员]，然后我们选择[修改,系统管理员]，就能看到下一步最大的可能还是[删除，系统管理员]，这与前面发掘出的频繁序列呼应。
还有类似的每步都可以预测的长序列：
[公文起草,测试]->[修改,测试]
[公文起草,测试]->[送办理,测试]->[送合成公文,测试]
